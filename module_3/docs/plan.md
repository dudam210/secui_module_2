# 시스템 모니터링 프로젝트 개발 계획

## 프로젝트 목표

단일 서버의 실시간 시스템 리소스(CPU, 메모리, 디스크, 네트워크)를 수집, 저장 및 시각화하는 모니터링 시스템 구축

## 개발 단계

### Module 1: 기본 모니터링 시스템
- [ ] JavaScript 기반 클라이언트 사이드 모니터링 구현
- [ ] 실시간 차트 시각화 구현 (chart.js)
- [ ] Node.js 백엔드 서버 구현
- [ ] 시뮬레이션 모드 구현
- [ ] 실제 시스템 데이터 수집 구현
- [ ] 반응형 대시보드 UI 완성
- [ ] 프로세스 목록 모니터링 추가

### Module 2: 프론트엔드 개선
- [ ] 바닐라 JavaScript 웹 애플리케이션 구조 설계
- [ ] CSS 스타일 시트 정리 및 개선
- [ ] 사용자 인터페이스 컴포넌트 개발
- [ ] 정적 리소스 관리 (assets)
- [ ] 빌드 없는 배포 환경 구성

### Module 3: 고급 백엔드 시스템
- [ ] Python FastAPI 기반 REST API 서버 구축
- [ ] psutil을 이용한 시스템 메트릭 수집기 구현
  - [ ] CPU 메트릭 수집기
  - [ ] 메모리 메트릭 수집기
  - [ ] 디스크 메트릭 수집기
  - [ ] 네트워크 메트릭 수집기
  - [ ] 프로세스 모니터 구현
- [ ] 시계열 데이터베이스 구현
  - [ ] InfluxDB 통합
  - [ ] SQLite 대체 옵션 구현
- [ ] 데이터 보관 정책 구현
  - [ ] 1초 해상도: 1일 보관
  - [ ] 1분 해상도: 7일 보관
  - [ ] 5분 해상도: 30일 보관
  - [ ] 1시간 해상도: 1년 보관
- [ ] APScheduler를 이용한 주기적 메트릭 수집
- [ ] API 엔드포인트 구현
  - [ ] `/api/v1/metrics/current` - 현재 메트릭
  - [ ] `/api/v1/metrics/cpu` - CPU 시계열 데이터
  - [ ] `/api/v1/metrics/memory` - 메모리 시계열 데이터
  - [ ] `/api/v1/metrics/disk` - 디스크 시계열 데이터
  - [ ] `/api/v1/metrics/network` - 네트워크 시계열 데이터
  - [ ] `/api/v1/metrics/processes` - 프로세스 목록
  - [ ] `/api/v1/health` - 헬스체크
- [ ] 알림 시스템 구현
  - [ ] 임계값 기반 알림 로직
  - [ ] 이메일 알림 구현
  - [ ] Slack 웹훅 알림 구현
  - [ ] 기본 알림 규칙 설정 (CPU >80%, 메모리 >90%, 디스크 >85%)

## 기술 스택

### 프론트엔드
- HTML5, CSS3, JavaScript (ES6+)
- 차트 라이브러리 (SimpleChart)
- 반응형 디자인

### 백엔드
- Node.js (Module 1)
- Python 3.10+ (Module 3)
- FastAPI
- psutil
- APScheduler

### 데이터베이스
- InfluxDB (시계열 데이터)
- SQLite (간단한 구현)

### 배포
- Docker
- Docker Compose

## 성능 목표
- 메트릭 수집 오버헤드: CPU <5%
- API 응답 시간: 평균 <200ms
- 동시 연결 지원: 100개 이상
- 메트릭 수집 주기: 1-60초 (설정 가능)
